A 5x5 section is drawn to the main level in real time, using levels that were already baked to acheive a procedural main level.

How I plan to do better

Ideas: add check to enable the one time spawning of special tiles and items. add checks to ensure that two of the same room don't spawn next to each other. add a third layer or array to create tiles on the z-axis. add more unique levels to increase different outcomes add target points that can communicated to each other, such as "locked doors and keys" add checks to ensure some levels can spawn together, such as adding doors linking with other doors add procedural elements into the arrayed levels themselves to add an extra layer. performance upgrades: add a loading screen to allow cpu to draw the random tiles to screen

Breakdown

The main variables

Level grid size refers to the grid size: any increase here would allow for larger randomized levels tiles size is the size of the actual tiles being generated Streaming level names is the specific array I am using to incorporate the levels And the random integer is just choosing a number from 0 to 4 muliplying it by 90 to get a random rotation of the tile

One step at a time

Putting it all together

We append the indexes of the 2 for loops and add that to a unique instanced name(the main level). we get the length of the array and add it as a targe to the create instance node. returned values of the x and y locations as well as the random rotation of the tile are set to the level transform.

Drawing to screen

From our Create Intance function we set the level transform with the values we calculated. We then set that to be loaded into the main level. The next thing we set is visibility within the level. From there you have a procedurally generated level, where you can create detailed levels that link together
