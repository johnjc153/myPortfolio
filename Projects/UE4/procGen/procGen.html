Breakdown

The main variables

  • Level grid size refers to the grid size: any increase here would allow for larger randomized levels.
  • Tiles size is the size of the actual tiles being generated.
  • The Streaming Level Names array is the specific array that contains the stored levels.
  • A random integer with a max index of 3, so four positions for four sides, is then multiplied by 90 to get the z (yaw) rotation. This spins the drawn segment to face a different direction.

It gets a bit complicated

On Event BeginPlay two for loops are called. One nested inside another, so each time the first one iterates (x) the second (y) does the same. Each loop has a default starting index of 0, and the last index is set to the level grid size variable. These then cast to the create instance node. The level grid size is then multiplied by the tile size to get the actual size of the tile and then is divided by two. This value is then used to subtract the value of the tile size multiplied by the index of each loop to get to transform of both the x and y locations of the arrayed objects. We take the X and Y locations and the Z (yaw) rotation data and cast it to the transform node. Taking the length of the streaming level names or segments array we generate a random integer and compare that to the actual array to get a streaming level.

Putting it all together

We append the indexes of the two for loops and add that to a unique instanced name(the main level). We get the length of the array and add it as a target to the create instance node. Grabbing the returned values of the x and y locations, as well as the random rotation of the tile z, we are set make the level transform.

Drawing to screen

From our Create Instance function, we set the level transform with the values we calculated. We then set that to be loaded into the main level. The next thing we set its visibility within the level. From there you have a procedurally generated level, where you can create detailed levels that link together.
