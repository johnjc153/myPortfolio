This is my Climb Toolkit

Climbing system in UE4, allows for the player to climb "climbable" objects using a series of checks and overlapping events.

What I would do better

  • Retargeting the animations would help to reduce clipping.
  • Adding an IK model to help snap limbs to the correct places on a climbable object.
  • Overhaul of certain variables and elements that combine markers and tracer checks to help improve performance by reducing the load of tracer checks.

Check Tracers

On Every Tick, we call our forward, height, and jump tracers, while constantly checking if we are hanging. In a sequence, we initialize our left and right tracers, our jump event, and our turning back and forward functions. Then we check if we can move or jump to the left as well as the right. We also check if we can turn a corner.

Check Logic

This is where we check if the player can move left or right using our arrow as a guide and a ledge trace channel.

Climb Ledge

The climb ledge event sets the character’s movement mode to “walking.” If we are not climbing the ledge we are in a climbing state and then we call the climb ledge function. The climb ledge function traces and then sets the movement mode to “flying.” After this is called we set Is Climbing Ledge to true and Is Hanging to false.

Exit Ledge

On the event call, we set the movement mode to “walking.” We call “can grab” and set Is Handing to false.

Jump

Here we check to see if we can jump right, jump left, jump up, as well as climb the ledge using the tracers.

Jump Up tracer

We Check if the player can jump up from a trace channel of our ledge trace

Jumping to the side

We check if the player can jump to the left and right. If there isn’t a return value from our trace channel we return Can Jump left or right to false.
